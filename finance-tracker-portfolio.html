Build Process

End-to-end pipeline

01
Setup
Folder Structure & Source Files
Created dedicated folders — one per bank for raw PDF statements, and a separate folder for the Excel working file. This keeps source data cleanly isolated from the transformation layer.
02
Ingestion
Get Data → From Folder (Power Query)
Used Data → Get Data → From Folder to point Power Query at the Scotia bank statement folder. Selected Transform Data to enter the query editor rather than loading directly, enabling full transformation control.
03
Extraction
Unpacking PDF Binaries
The folder query surfaces PDF files as Binary values in the Content column. Used a custom column formula to extract all tables from each PDF binary:
= Pdf.Tables([Content])
04
Filtering
Isolating the Right Tables
Expanded the custom column and used the kind column to keep only table rows (filtering out pages). A second custom column counted columns per table — only tables with exactly five columns were retained, matching the transaction layout.
05
Headers
Promoting Headers
Data previews revealed that the first row of each table held the column names. Created a custom column to promote headers on each nested table:
= Table.PromoteHeaders([Data])
06
Shaping
Final Column Selection & Expansion
Kept only the PDF filename column and the promoted-headers column, then expanded the latter — excluding the Balance column — to arrive at five clean columns: Name, Date, Transaction Details, Debit, Credit.
07
Cleaning
Removing Noise Rows
Filtered out summary rows that carry no transactional value — specifically "Closing Balance" and "Balance Brought Forward" entries in the Transaction Details column. Debit and Credit nulls were also cleared.
08
Multi-bank
TD Bank Statements
Repeated the same pipeline for the TD folder. One additional step was required: dashes in the Withdrawals and Deposits columns were replaced with null before casting to a currency type — ensuring clean numeric values across both sources.
09
Merge
Append Queries → All Banks
Appended the Scotia and TD queries as a new query named All_Banks. Loaded it as a connection-only query and added it to the data model — laying the groundwork for DAX measures and KPI calculations.
10
Intelligence
Category Table & Fuzzy Keyword Matching
Built a Category reference table with three columns: Keyword, Category, and Importance (Need / Want / Savings). A custom M formula in All_Banks performs case-insensitive fuzzy matching against every keyword, assigning categories automatically — or "Others" when no match is found:
= let search = Text.Upper(Text.Remove([Transaction Details], {" ", ".", "-", "_"})), match = Table.SelectRows( Category_List, (r) => let key = Text.Upper(Text.Remove(r[Keyword], {" ", ".", "-", "_"})) in Text.Contains(search, key) ) in if Table.RowCount(match) = 0 then "Others" else match{0}[Category]
